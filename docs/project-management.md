# Gestión del proyecto

## Tablero Trello
[Ver tablero en Trello](https://trello.com/b/1TJHDhGh/gastosapp)

## Metodología
Para este proyecto se usa **Kanban** porque el desarrollo es individual
y las prioridades pueden cambiar durante el proceso. El tablero tiene
cinco columnas: Backlog, Todo, In Progress, Review y Done.

## Flujo de trabajo
1. Las funcionalidades nuevas entran en **Backlog**
2. Cuando se va a empezar una tarea se mueve a **Todo**
3. Durante el desarrollo se mueve a **In Progress**
4. Al terminar pasa a **Review** para comprobar que funciona
5. Si todo está correcto se mueve a **Done**

## Organización del repositorio
- `src/` — código fuente del frontend
- `server/` — código fuente del backend
- `docs/` — documentación del proyecto
